## 1. ゲームの目的と勝利条件

2人のプレイヤーがそれぞれ60枚のデッキを使い、ポケモンで対戦するカードゲーム。
以下の **3つのうちいずれか** を達成したプレイヤーが勝利する。

| # | 勝利条件 | 説明 |
|---|---------|------|
| 1 | サイドを全部取り切る | 相手のポケモンをきぜつさせてサイドカードを6枚すべて取る |
| 2 | 相手の場にポケモンがいなくなる | 相手のバトルポケモンがきぜつし、ベンチにもポケモンがいない |
| 3 | 相手がカードを引けない（山札切れ） | ターン開始時に山札が0枚で1枚引けない場合、そのプレイヤーの敗北 |

## 2. カードの種類

### ポケモンカード
バトルの主役。ワザを使って相手のポケモンを攻撃する。

| サブタイプ | 説明 |
|-----------|------|
| **たねポケモン** | 手札からベンチに直接出せる基本のポケモン |
| **1進化ポケモン** | 場のたねポケモンの上に重ねて進化させる（出したターンには進化不可） |
| **2進化ポケモン** | 場の1進化ポケモンの上に重ねて進化させる |
| **ポケモンex** | HPが高く強力だが、きぜつ時にサイドを **2枚** 取られる |
| **テラスタルポケモンex** | ベンチにいる限りワザのダメージを **一切受けない**。きぜつ時サイド2枚 |
| **メガシンカポケモンex** | 非常に高いHP。きぜつ時にサイドを **3枚** 取られる |

### トレーナーズカード
様々な効果を持つサポートカード。

| サブタイプ | 使用制限 | 説明 |
|-----------|---------|------|
| **グッズ（アイテム）** | 1ターンに何枚でも | 使用後はトラッシュ |
| **サポート** | **1ターンに1枚だけ** | 強力な効果。先攻1ターン目は使えない |
| **スタジアム** | 1ターンに1枚 | 場に出して両プレイヤーに効果（後述） |
| **ポケモンのどうぐ** | 何枚でも（各ポケモンに1枚） | ポケモンにつけて使う（後述） |
| **ACE SPEC** | **デッキに1枚のみ** | 非常に強力な特別カード |

### エネルギーカード
ポケモンのワザを使うために必要。

| サブタイプ | デッキ制限 | 説明 |
|-----------|-----------|------|
| **基本エネルギー** | **枚数制限なし** | 草・炎・水・雷・超・闘・悪・鋼の8種類 |
| **特殊エネルギー** | 同名4枚まで | 特別な効果を持つエネルギー |

## 3. デッキ構築ルール

| ルール | 内容 |
|-------|------|
| デッキ枚数 | **ちょうど60枚** |
| 同名カード制限 | 同じ名前のカードは **4枚まで** |
| 基本エネルギーの例外 | 基本エネルギーは同名制限なし（何枚でもOK） |
| ACE SPEC制限 | デッキにACE SPECカードは **種類を問わず1枚まで** |
| たねポケモン必須 | デッキにはたねポケモンが **最低1枚** 必要 |

### 本コンペ固有の制限
- 使用可能カードは **ID 1〜1267** のみ（Card_ID_List_JP.pdf の39ページまで）

## 4. ゲームの準備（セットアップ）

1. **先攻・後攻決定**: コイントスで決定。勝ったプレイヤーが選択
2. **シャッフル**: デッキをシャッフルし、相手にカットさせる
3. **手札**: デッキの上から **7枚** を手札として引く
4. **バトル場**: 手札からたねポケモンを1体、裏向きでバトル場に出す
5. **ベンチ**: 手札からたねポケモンを任意でベンチに出す（最大5体）
6. **サイド**: デッキの上から **6枚** を裏向きにサイドとして置く
7. **開始**: バトル場とベンチのポケモンを表にしてゲーム開始

### マリガン（手札にたねポケモンがない場合）

1. 手札を相手に公開する
2. 手札をデッキに戻してシャッフル
3. 7枚引き直す
4. **相手** はマリガン1回につきカードを1枚引く権利を得る（任意）
5. たねポケモンが出るまで繰り返す

## 5. ターンの流れ

```
ドロー（必須）→ メインフェーズ（任意）→ ワザ（任意、使うとターン終了）→ ポケモンチェック
```

| フェーズ | 内容 | 必須？ |
|---------|------|--------|
| **ドロー** | デッキの上から1枚引く。引けなければ敗北 | 必須 |
| **メインフェーズ** | 各種アクションを任意の順番で行う（次項参照） | 任意 |
| **ワザ** | バトルポケモンのワザを1つ使う → ターン終了 | 任意 |
| **ポケモンチェック** | ターン間に行われる特殊状態の処理 | 自動 |

### ポケモンチェック（ターン間処理の順序）
1. **どく**: ダメカン1個のせる
2. **やけど**: ダメカン2個のせる → コインを投げ表なら回復
3. **ねむり**: コインを投げ表なら回復
4. **まひ**: 自分のターンを経過していれば回復

---

## 6. メインフェーズでできること

| アクション | 制限 | 説明 |
|-----------|------|------|
| エネルギーをつける | **1ターンに1枚** | 手札からポケモン（バトル場・ベンチ問わず）につける |
| たねポケモンをベンチに出す | ベンチ空きがあれば何体でも | 手札からたねポケモンをベンチに出す |
| ポケモンを進化させる | 下記の制限あり | 場のポケモンの上に進化カードを重ねる |
| グッズを使う | 何枚でも | 使用後トラッシュ |
| サポートを使う | **1ターンに1枚** | 使用後トラッシュ |
| スタジアムを出す | **1ターンに1枚** | 場に同名がある場合は出せない |
| 特性を使う | カードテキストに従う | ポケモンが持つ特性の効果を使う |
| にげる（リトリート） | **1ターンに1回** | にげるコスト分のエネルギーをトラッシュしベンチと入れ替え |
| ポケモンのどうぐをつける | 1体に1枚まで | グッズの一種 |

### 進化の制限
- そのポケモンが **場に出たターンには進化できない**
- **ゲーム最初のターン**（お互い）には進化できない
- 1ターンに複数のポケモンを進化させることは可能（それぞれ異なるポケモン）

### にげるの制限
- **ねむり** / **まひ** 状態のポケモンはにげられない
- ベンチにポケモンがいなければにげられない

## 7. ワザの使い方とダメージ計算

### ワザの使用条件
- バトルポケモンについているエネルギーが、ワザに記載されたコスト以上であること
- **タイプ指定エネルギー**: 指定されたタイプが必要
- **無色エネルギー（★マーク）**: どのタイプでも充当可能
- エネルギーはつけたまま（トラッシュはカードテキストで指定された場合のみ）

### ダメージ計算の順序
1. ワザの基本ダメージを決定
2. 効果によるダメージ増減を適用（どうぐ、スタジアム、特性等）
3. **弱点を適用（×2）**
4. **抵抗力を適用（-30）**
5. 最終ダメージが0未満の場合は0とする

---

## 8. 弱点と抵抗力

### 弱点
- ポケモンカードの左下に記載
- 攻撃するポケモンのタイプが弱点のタイプと一致する場合に適用
- **現行ルール: ダメージ ×2**

### 抵抗力
- ポケモンカードの左下（弱点の隣）に記載
- すべてのポケモンが持つわけではない
- **現行ルール: ダメージ -30**

### 適用順序
**弱点（×2）→ 抵抗力（-30）** の順で計算する

### 計算例
攻撃ダメージ50、相手が弱点の場合: 50 × 2 = 100
攻撃ダメージ50、相手が抵抗力を持つ場合: 50 - 30 = 20
攻撃ダメージ10、相手が抵抗力を持つ場合: 10 - 30 = 0（マイナスにはならない）

## 9. 特殊状態（状態異常）

**バトルポケモンのみ** に適用される5種類の状態異常。

### 共通の回復方法
- ベンチに下がる（にげる・カード効果で入れ替え）
- 進化する
- 特定のカード効果で回復

### 相互関係
- **ねむり・まひ・こんらん** はカードの向きで表現するため、互いに **上書き** される
- **どく・やけど** はマーカーで示し、他の状態と **併存可能**

| 状態 | 効果 | 表示方法 | 回復方法 |
|------|------|---------|---------|
| **どく** | ポケモンチェックのたびにダメカン1個 | どくマーカー | ベンチ移動/進化/カード効果 |
| **やけど** | ポケモンチェックでダメカン2個→コイン表で回復 | やけどマーカー | コイン表/ベンチ移動/進化 |
| **ねむり** | ワザ使えない・にげられない | カードを反時計回りに横向き | コイン表/ベンチ移動/進化 |
| **まひ** | ワザ使えない・にげられない（次の自分ターン終了まで） | カードを時計回りに横向き | 1ターン経過で自動回復/ベンチ移動/進化 |
| **こんらん** | ワザ使用時コイン→裏なら失敗＋自分にダメカン3個 | カードを上下逆さま | ベンチ移動/進化/カード効果 |

## 10. きぜつとサイド（プライズ）

### きぜつの条件
ポケモンのHP以上のダメカンがのった場合。

### きぜつ時の処理
1. きぜつしたポケモンとついているカード（エネルギー・どうぐ等）を **すべてトラッシュ**
2. 相手プレイヤーが **サイドからカードを取る**
3. ベンチからポケモンを **バトル場に出す**（ベンチにポケモンがいなければ敗北）

### サイドの取り枚数

| ポケモンの種類 | きぜつ時に取られるサイド |
|--------------|----------------------|
| 通常のポケモン | **1枚** |
| ポケモンex | **2枚** |
| メガシンカポケモンex | **3枚** |

---

## 11. ポケモンex / メガシンカex / テラスタル

### ポケモンex
- HPが通常のポケモンより高い
- きぜつ時にサイドを **2枚** 取られる
- たね・1進化・2進化のいずれも存在し、通常の進化ルールに従う

### テラスタルポケモンex
- ポケモンexの一種
- **ベンチにいる限り、ワザのダメージを一切受けない**（自分のワザからも）
- 通常と異なるタイプを持つ場合がある
- きぜつ時にサイドを **2枚** 取られる

### メガシンカポケモンex
- 非常に高いHP（ゲーム中最高クラス）
- きぜつ時にサイドを **3枚** 取られる
- 通常の進化ルールに従う

## 12. 先攻・後攻の違い

| 項目 | 先攻 | 後攻 |
|------|------|------|
| ワザ | **最初のターンは使えない** | 最初のターンから使用可能 |
| サポート | **最初のターンは使えない** | 最初のターンから使用可能 |
| グッズ・進化等 | 使用可能 | 使用可能 |

先攻の利点: 先にポケモンを展開・進化できる
後攻の利点: 最初のターンからワザ・サポートを使える

---

## 13. ベンチの上限

- バトル場に **1体**、ベンチに **最大5体** の計6体が場に出せる上限
- 一部のカード効果によりベンチ枠が増減する場合がある

---

## 14. スタジアムのルール

| ルール | 内容 |
|-------|------|
| 配置場所 | バトル場の横。場に **1枚のみ** |
| 効果範囲 | **両プレイヤー** に適用される |
| 持続 | 場に出ている限り効果が続く |
| 使用制限 | 1ターンに1枚。場に同名スタジアムがある場合は出せない |
| 除去方法 | 別のスタジアムを出す（前のはトラッシュ）/ カード効果 |
| 先攻1ターン目 | **出せる**（サポートの制限とは別） |

---

## 15. ポケモンのどうぐのルール

| ルール | 内容 |
|-------|------|
| 分類 | トレーナーズカードのグッズ（アイテム）の一種 |
| 装着対象 | 自分のポケモン（バトル場・ベンチ問わず） |
| 枚数制限 | **1体のポケモンにつき1枚まで** |
| 持続 | きぜつするかカード効果で外されるまで場に残る |
| 取り外し | 通常は自由に外せない。カード効果でのみ外せる |
| きぜつ時 | ポケモンがきぜつしたら、ついていたどうぐもトラッシュ |

## 16. シミュレータ（cabt）と公式ルールの差分

> **重要: 本コンペでは「シミュレータの挙動が正」として扱われる。**

### 差分① 一部のワザが選択不可になる

| 公式ルール | シミュレータ |
|-----------|------------|
| 攻撃を宣言→効果が解決できずターン終了 | そのような攻撃は最初から選択肢に含まれない |

該当例:
- ベンチに空きがないのに「デッキからたねをベンチに出す」効果のワザ
- デッキが0枚なのに「カードを引く」効果のワザ
- 相手の手札が0枚なのに「相手の手札に作用する」効果のワザ

→ 最終結果は同じ。影響は軽微。

### 差分② メガジガルデexの「ゼロをなくすもの」

| 公式ルール | シミュレータ |
|-----------|------------|
| プレイヤーがターゲット順を選べる | 左から右に自動でコインフリップ |

→ きぜつ処理は同時なので影響なし。

### 差分③ 両者同時きぜつ時のサイド取得順

| 公式ルール | シミュレータ |
|-----------|------------|
| 次ターンのプレイヤーが選ぶ→相手が選ぶ→**同時に取る** | 次ターンのプレイヤーが選んで**取る**→相手が選んで**取る** |

→ 両者が取り切ると引き分け扱いなので影響なし。

### 補足（Q&Aより）
- エースバーン（カードID 666）は特性により対戦準備時にバトル場に出すことが可能
- 参加者はDataタブのカードリストに基づいて自分のデッキを構築可能

## 17. cabt API の主要構造

### エージェントの基本形
```python
def agent(obs_dict: dict) -> list[int]:
    # obs_dict["select"] が None → 初期デッキ選択フェーズ。60枚のカードIDを返す
    # それ以外 → obs["select"]["option"] の中から選んだ index を返す
    return [...]
```

### Observation の3つの柱
| フィールド | 説明 |
|-----------|------|
| `logs` | 前回の選択以降に起きたイベント列 |
| `current` | 現在の盤面状態（State）。初期デッキ選択時は None |
| `select` | 選択肢情報（SelectData）。初期デッキ選択時は None |

### SelectContext（何を選んでいるか）の主要値
| ID | 名前 | 説明 |
|----|------|------|
| 0 | MAIN | メイン行動選択 |
| 1 | SETUP_ACTIVE_POKEMON | 対戦準備: バトル場にポケモン配置 |
| 2 | SETUP_BENCH_POKEMON | 対戦準備: ベンチにポケモン配置 |
| 3 | SWITCH | 入れ替えるポケモン選択 |
| 35 | ATTACK | 使うワザ選択 |
| 37 | EVOLVE | 進化先選択 |
| 41 | IS_FIRST | 先攻にするか？ |
| 42 | MULLIGAN | 手札を引き直すか？ |

### OptionType（選択肢の種類）の主要値
| ID | 名前 | 説明 |
|----|------|------|
| 7 | PLAY | 手札からカードを出す |
| 8 | ATTACH | エネルギー/カードをつける |
| 9 | EVOLVE | 進化させる |
| 10 | ABILITY | 特性を使う |
| 12 | RETREAT | にげる |
| 13 | ATTACK | ワザを使う |
| 14 | END | ターン終了 |

### 勝利判定（LogType.RESULT）
- `result = 0` → プレイヤー0の勝利
- `result = 1` → プレイヤー1の勝利
- `result = 2` → 引き分け
- `reason`: 1=サイド0枚, 2=山札0枚でターン開始, 3=バトル場にポケモンなし, 4=カード効果
