## サイクル1：エネルギー削減＋サーチ/加速（変種A）

  （このデッキはトラッシュにエネが乗りにくいため、寄与が薄い可能性）。

### Act（次サイクルへの示唆）
1. **打点強化（goal2）を最優先**：増えた引き分け（特にVenusaur V2の16分け）を勝ちに変える。

2. **非ex速攻（Iono / Alakazam）対策**：特性が効かず構造的に不利。デッキだけでは限界 → **②モデル改良**または別プランで対処。
3. **同一フィールドBase差分の確定**：純改善幅を数値化（計測完了待ち）。
4. 力の砂時計の寄与を検証：トラッシュにエネが乗りにくいデッキ特性のため、効果が薄ければ枠を打点/サーチに振り替える。

## サイクル2：打点＝詰め強化（変種B＝ボスの指令）

### Plan
- 仮説：壁は生存できるが倒し切れず引き分け化。**ボスの指令**でベンチを引きずり出し、
  削れた/低HP/高価値の的を狙撃KOして**サイドを取り切る（詰める）**＝打点(goal2)を補う。
- 構成：オリジナルBaseに **ボスの指令×3**（基本草19→16）。他はBaseと同一＝ボスの効果を単独で計測。
- スコアリング：Crustleが攻撃可能な時、相手ベンチに「今のバトル場より美味しい(KO/高サイド)的」が
  いれば高評価でボスを使用。gust対象は target_score で狙撃価値最大の相手を選択。

### Check（決定化＋ペア比較・同一シード・10試合/ペア）
| | 勝率 | 内訳 |
|---|---|---|
| Base | 65% | 599W-290L-31D |
| 変種B | 66% | 607W-282L-31D |
| 差 | **+1pt** | 勝ち+8 / 負け−8 / **分け±0** |

改善17 / 悪化15 / 同13体。

### 所見（重要）
- **ボスの指令はほぼ無効（+1pt＝ノイズ域）。狙いの「引き分け削減」は分け±0で未達。**
- 原因：引き分けの正体は**高HP ex をグレートシザー120で倒し切れない**こと。ベンチも同じ高HP ex
  なので、ボスで引きずり出してもKOが生まれない＝**「的選び」では足りない“1発の火力”を補えない**。
- 計測の**決定化に成功**：Baseが再現性高く65%で安定（旧69%はエージェント順ブレ由来のノイズ）。
  シードを「相手＋手番＋試合番号」依存にしてBaseと変種Bを同一シャッフルで戦わせるペア比較を実装。

### 総括（サイクル1〜2）
- 変種A(エネ削減＋サーチ)＝Base±ノイズ、変種B(ボス)＝Base+1pt。**デッキ小改良では天井(~65-66%)を超えない。**
- 壁の限界は構造的：①高HP ex を120で倒し切れない(分け) ②非exに特性が効かない(負け)。

### Act（次サイクル候補）
1. **変種C＝スグリ(ex/Vへ+30→150)**：分けの主因「高HP ex」に直接効く唯一の火力UP。
2. **モデル(方策)改良へ転換**：デッキは天井近い。探索・相手読み等。
3. 打ち止め（~65%がCrustleの実力と判断）。

## Web調査：競技イワパレスは「mono壁」ではなく「ハイブリッド」（2026-06）

シティリーグ入賞31・優勝1の実績デッキは、Crustle単体ではなく**殴れるアタッカー/ドローエンジンと分業**：
- **メガガルーラex(756)**：たね・HP300・特性おつかいダッシュ(毎ターン2ドロー)・マシンガンコンボ(無●●●で200+)。ドローエンジン＋壁＋フィニッシャー。
- **オーガポンいしずえのめんex(117)**：特性いしずえのかまえ＝特性持ちアタッカーの攻撃を無効（非ex/特性アタッカー対策）。闘●●で140(弱点/効果無視)。
- **マシマシラ(112)**：特性アドレナブレイン＝悪エネ付きでダメカン移動（Crustleの傷を相手へ移す＝疑似回復＋チップ）。
- tech：ポケギア3.0(1122)、ロケット団のラムダ(1219)、暗号マニアの解読(1188)。ACE SPEC ヒーローマント採用率100%（一致）。
- **すべて競技プール(in_engine=1)に存在＝構築可能。**
- 出典: ポケざんまい/ポケカブック/ポケカ千日手ノート/Limitless/Deltia's Gaming。

→ mono-Crustleの小改良が天井(~65%)だったのは構造的。メタは「Crustle=壁／メガガルーラ=攻撃＆ドロー」の分業で超えている。

## サイクル3：ハイブリッド（変種C＝Crustle＋メガガルーラex×2）

### Plan / Do
- Base＋メガガルーラex×2（基本草19→17）。スコアリング拡張：メガを場に出す/エネ付与/おつかいダッシュ使用/
  マシンガンコンボ評価/「ex相手はCrustle壁優先・非ex相手はメガを前に」の入替判断、target_scoreを打点引数化(120/200)。

### Check（ペア比較・同一シード・10試合/ペア）
| | 勝率 | 内訳 |
|---|---|---|
| Base | 66% | 619W-294L-27D |
| 変種C | **59%** | 559W-348L-33D |
| 差 | **−7pt** | 勝ち−60 / 負け+54 |

- **悪化**：ドラパルトex 100→50、メガルカリオex 85→50、ミラー 90→60（ex対面を自壊）。
- **改善**：Crustle Bot +50、anti-wall +15（mono-Crustleの弱点=同系/非exには効いた）。

### 所見
- **naïveなメガガルーラ投入は逆効果**：メガは特性無効が無く普通に殴られ、倒されるとサイド3枚。v1の操縦では
  壁Crustleの代わりに晒されて溶け、**得意だったex完封を失う代償**が、ミラー改善を上回った。
- 方向性（ハイブリッド）は正しい（改善対面が狙い通り）が、**操縦が未熟**＝「exにメガを絶対晒さない／安全時のみ前に出す／メガは倒されない立ち回り」を作り込む必要。
- メガは benched だと おつかいダッシュ(要active)も攻撃(要active)も不可＝置物化（実測: 攻撃0.09回/試合）。さらにベンチのメガは相手ボスの3枚サイド標的になる。

### 変種C v2（ex主体マッチではメガガルーラを封印する操縦）
- 実装：`opponent_has_ex()` で相手の場のex/メガexを検出。ex主体なら**メガを場に出さない・前に出さない・サーチしない**（純Crustle壁）。非exの時だけメガを展開。
- 結果（ペア比較・10/pair）：Base 65% vs **変種C v2 61%（−4pt）**。v1(−7pt)から改善も、まだマイナス。
- 改善：anti-wallメガルカリオ+30、イワパレス対応メガルカリオ+30、Crustle Bot+25、Venusaur+15（狙い通り）。
- 悪化：ドラパルトex 100→65/70、ミラー 100→70、Tiny RL 100→70（**ex対面が依然regress**）。
- **根本原因＝「後出しでex判明」問題**：序盤は相手がexか判別不能。非exと誤認してメガを展開→後からexが出ると、メガが3枚サイドの置物＋Crustle展開遅延。後付けゲートでは防げない。

### Act（結論）
- **メガガルーラの“後付け”はex対面（多数派・元100%）を必ず多少削り、改善分を上回る**（v1 −7 → v2 −4、0を超えない見込み）。
  mono-Crustleの強さ＝無条件のex完封を、メガ投入が壊すため。後付けパッチの限界。
- **デッキ路線の結論：mono-Crustle（65-66%）が現実的天井。** ハイブリッドはメタで強いが、別シェル＋専用操縦の
  “ground-up再構築”が前提＝大コスト・不確実。費用対効果で次を判断：
  1. ハイブリッドをゼロから再構築（大・不確実）
  2. mono-Crustle確定→モデル(方策)改良へ
  3. 緑単カミツオロチ等の別アーキタイプを調査

### 現時点の到達点まとめ
| デッキ | 対フィールド | vs Base |
|---|---|---|
| Base(mono-Crustle) | 65-66% | — |
| 変種A(エネ削減＋サーチ) | ~65-68% | ±ノイズ |
| 変種B(ボス＝詰め) | 66% | +1pt |
| 変種C(メガガルーラ ハイブリッド v1) | 59% | **−7pt** |
→ **デッキ路線は mono-Crustle が天井。超えるにはハイブリッドの“操縦”作り込み or モデル改良が必要。**

## サイクル4：探索エージェント（MCTS-PIMC）へ転換 ★着手

デッキ路線が天井(~66%)と判明したため、「強いデッキ（ハイブリッド）＋それを操縦できる探索エージェント」へ。
- **判定：MCTS(PIMC) を採用**（学習不要・既存ヒューリスティックをrollout流用・先読みで操縦獲得）。RL(AlphaZero)は過剰/重い、PPOサンプルは看板倒れ。
- **環境**：`cg/api.py` に専用 Search API（`search_begin/search_step/search_release/search_end`）。本戦`battle_ptr`と別の`agent_ptr`上で前方シミュレーション、隠れ情報はPIMC決定化で補う。
- **PoC成功（2026-06-21）**：前方シミュレーション・分岐・深掘り・破棄・本戦非干渉すべてOK（`forward_sim=OK branch=True deeper=True real_unaffected=True`）。最大リスク解消。
- 詳細設計：**[[mcts_agent_design]]**（`docs/mcts_agent_design.md`）。
- **実証PoC2成功（2026-06-21）**：最小MCTS（変種Cの評価をrollout流用, mirror相手モデル, depth80）が、
  同一デッキ head-to-head で**ヒューリスティック単体に 16勝4敗(80%・p≈0.6%)** で勝利。「探索操縦＞greedy」を実証。
  教訓：相手の決定化と rollout 深さが質を左右（無害相手・浅探索は逆効果）。速度~4.3s/試合(エミュ)。
- 次：**対フィールド勝率で検証**（重いのでEC2ネイティブx86が有効）→ 決定化数/sim数/時間予算チューニング。

## メモ：計測の実施方法
- 個別AI評価（1体 vs 全体）：`tools/run_tournament.py --challenger <id> --games-per-pair 10`
- 本番(VPS)はネイティブで高速だが gunicorn 1worker/2vCPU のため計測中はサイトが重くなる。
- 計測対象（登録AI）= `data/user_agents/`。VPSへは追加rsyncのみ（VPS固有エージェントを消さない）。
